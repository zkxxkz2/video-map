# 📱 iOS 满天星内容分发系统 (iOS Edge Delivery Engine) 架构方案

## 1. 架构定位与核心目标
本架构专为 **1000台非越狱 iOS 设备** 设计的极简、纯内网短视频分发中枢。这套系统将作为 `MoneyPrinterTurbo` 等批量生成工具的“下游发卡平台”。
核心目标以最低的硬件投入和开发成本，实现：
1. **千机千面**：确保 1000 台设备每次获取到的视频具备绝对唯一性（从文件 Hash 到视觉内容防重），规避平台查重与限流。
2. **可视化与双轨分发**：提供 PC 端的中心化监控大盘，同时在手机端提供底层 API 之外的 H5 可视化备用提货入口。
3. **纯本地化运行**：数据零云端交互，完全在宿主机本地和局域网闭环，保障资产绝对安全。

---

## 2. 硬件与局域网物理拓扑
*   **中央调度主机**：1 台跑在局域网固定 IP（如 `192.168.1.100`）的 Mac Mini 或 Windows PC（大容量 SSD 固态硬盘）。
*   **局域网无线矩阵**：
    *   主机通过**千兆有线网卡**直连核心路由交换层。
    *   手机矩阵连接至多台企业级无线 AP，通过 5GHz 频段确保内网分发速度。

---

## 3. 中央服务端设计 (单机 Web 中枢)
建议技术栈：`Python (FastAPI + Jinja2 + Uvicorn)` 或 `Node.js (Express)`，使用内置 `SQLite`。

### 3.1 本地存储级防查重机制
1.  **原始素材池 (`Asset_Pool`)**：接收 `Video-Map` 等工具自动输出的批量 MP4 视频。
2.  **已分发隔离池 (`Archived`)**：一旦完成分发，被领走的视频文件在物理层面上被移动到该目录，这能以 100% 的确定性防止内容重复下发。
3.  **动态 Hash 洗稿隔离（关键动作）**：接口在发送文件流的瞬间，在尾部注入随机字节并剥离原始 EXIF，确保 1000 个设备的源文件即使非常类似，物理 MD5 等特征码也截然不同。

---

## 4. 全方位双轨分发机制与可视化 WebUI

为满足可靠性与实时管控，系统将搭载 Web 服务提供三个维度的界面访问：

### 4.1 PC 主控端：大盘可视化 Web 面板 (Master Dashboard)
*   **访问方式**：在主机浏览器打开 `http://127.0.0.1:8000/admin`。
*   **功能规划**：
    *   **库存水文监测**：实时显示现有可用短视频量（如 `可用 1500 / 警告线 500`），低于阈值可自动发声报警提醒人为干预（继续用 Video-Map 生成）。
    *   **分发进度实时墙**：一个包含 1000 个小圆点/方块的矩阵图。绿色代表今日已成功提货（如 `iPhone_001`），灰色代表尚未提货，红色代表设备多次请求失败或拉取异常。
    *   **人工撤回与溯源**：查阅派发台账，随时清楚某一特定视频（如发生违规）到底是被哪台手机发出去的。

### 4.2 手机端：H5 备用分发页 (Mobile Web Portal Fallback)
为了防止部分 iOS 设备快捷指令（Shortcuts）闪退或配置异常导致当天没货可发，必须设立网页端降级方案。
*   **访问方式**：手机在局域网内用 Safari 浏览器打开 `http://192.168.1.100:8000/mobile`。
*   **功能规划**：
    *   **设备身份固化 (Cookie/LocalStorage)**：首次打开输入设备号 `iPhone_XXX`，之后该设备的 Safari 永久记住身份，无需每次输入。
    *   **极简按钮模式**：页面正中间只有一颗大按钮 —— **“获取今日视频”**。
    *   **结果反馈**：点击后直接从浏览器下载 MP4，弹出“已成功领走 1 个视频，编号 XX”，用户手动将其按保存至系统相册。

### 4.3 效率巅峰：iOS 快捷指令自动化 (Primary Automation)
*   首选自动流为**无感化**快捷指令。结合系统“个人自动化（如每天上午 10:00 执行）”访问底层 API。
*   发送请求 `GET /api/fetch_video?device_id=XXX`，接口返回视频数据流。
*   利用快捷指令内的 `[存至相册]` 动作，在后台静默将干干净净（已洗稿）的视频放进手机里待人工稍后发布。

---

## 5. 落地规划与系统解耦优势

将内容生成引擎(`MoneyPrinterTurbo`) 和 这个 `iOS 分发系统` 物理分离，是非常健康的设计：
1. **生成端** 专门拼算力，专注于日产千篇高质量视频。
2. **分发端** 专门做网关，专注于管理这上千台手机的高并发拉取、防重及 Hash 隔离。
3. 通过一个公共的本地共享文件夹连通两者，哪怕其中一台机器关机维修，另一边也不会受影响失效。
